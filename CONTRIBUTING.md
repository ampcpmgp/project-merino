# Contributing to Project Merino

## 開発ガイドライン (Development Guidelines)

### コーディング規約 (Coding Standards)

1. **コードの品質**
   - クリーンで読みやすいコードを書く
   - 共通処理は適切にリファクタリングする
   - 変数名と関数名は意味が明確なものを使用する

2. **イベントハンドリング**
   - イベントリスナーは適切にクリーンアップする
   - ESCキーなどのキーボードイベントは適切に管理する
   - ダイアログのスタック管理を確実に行う

3. **UI/UX**
   - ユーザーインターフェースは直感的で使いやすくする
   - ダイアログは階層的に管理し、一つずつ確実に閉じる
   - 不要な機能は削除し、必要な機能は確実に動作させる

4. **メンテナビリティ**
   - 新しいダイアログやモーダルを追加する際も、既存のパターンを踏襲する
   - ドキュメントコメントを適切に記述する
   - VERSION_INFOは必ず更新する

### バージョン管理

プロジェクトに変更を加える際は、以下を必ず更新してください：

1. **VERSION_INFO** - バージョン番号と変更内容を記載
2. コミットメッセージは変更内容を明確に記述

### テスト

- 実装後は必ず動作確認を行う
- エッジケースも考慮する
- 既存機能が壊れていないか確認する

### コードレビュー

- プルリクエストは小さく、レビューしやすい単位で作成する
- 変更の意図を明確に説明する
- レビューコメントには誠実に対応する
